AI와 함께하는 효율적인 개발 방법론

체계적인 AI 협업 가이드

🔮 핵심 개념: 엔트로피(Entropy)

엔트로피란 무엇인가요?

엔트로피 = 혼란도, 불확실성, 모호함의 정도

AI가 코드를 생성할 때 발생하는 예측 불가능성을 의미합니다.

😵

높은 엔트로피

모호한 지시
"적절히 처리해줘"

😌

낮은 엔트로피

명확한 지시
"에러 시 로그 기록 후
'오류 발생' 메시지 표시"

Task Entropy (크게) - Prompt Entropy (작게) = 유용한 정보

📝 실제 예시

❌ "이 기능 좀 최적화해줘" → AI가 임의로 해석

✅ "실행시간을 100ms 이하로, 인덱스 활용과 서브쿼리 제거 우선" → 명확한 결과

📊 정보 이득(Information Gain)

정보 이득이란?

새로운 정보를 추가했을 때 불확실성이 줄어드는 정도

IG(T,a) = H(T) - H(T/a)
정보 이득 = 이전 엔트로피 - 정보 추가 후 엔트로피

모호한 상태

불확실성 높음

→

정보 추가

컨텍스트 제공

→

명확한 상태

불확실성 낮음

✅ 정보 이득 임계점

충분한 정보가 모이면 실행 가능한 상태가 됩니다.

임계점 도달 전에는 절대 실행하지 마세요!

🎭 AI 할루시네이션의 3가지 유형

할루시네이션 = AI가 사실이 아닌 내용을 그럴듯하게 만들어내는 현상

1

모델 해상도 부족

AI 모델 자체의 한계로 인한 오류

예: "STRAWBERRY의 R 개수 세기" → 토큰 단위로 처리되어 정확히 못 셈

해결: 외부 도구나 검증 시스템 활용

2

온도 설정으로 인한 변동성

창의성을 위한 랜덤성이 오류를 유발

예: 코드 생성 시 불필요한 창의적 변형

해결: 코딩 작업 시 낮은 온도 설정

3

입력 정보 부족

불충분한 컨텍스트로 인한 추측과 가정

예: "세종대왕이 맥북을 던진 사건" → 존재하지 않는 조합

해결: 충분하고 명확한 컨텍스트 제공

🔍 정보 부족의 3가지 패턴

📚

Knowledge Gap

모델 학습 데이터에
없는 정보

→ 외부 문서/DB 제공
📁

Context Gap

필요한 파일/코드/
설정 부족

→ 전체 코드 제공
📝

Specification Gap

"알아서 처리해줘"
같은 모호한 지시

→ 명확한 요구사항

⚠️ 위험 신호 (정보 부족 상태)

  • "일반적으로", "아마도" 같은 추측형 단어 사용
  • 구현 대신 설명이나 여러 옵션 제시
  • 3줄 이상의 주석으로 로직 설명

✅ 안전 신호 (실행 가능 상태)

  • 즉시 구체적인 코드 생성
  • 파일명, 함수명 정확히 참조
  • 에러 메시지 구체적 인용

🎯 원칙 1-3: 정보 관리의 기초

1

엔트로피 최소화

작업의 불확실성을 줄여 예측 가능한 결과 도출

  • 모호한 표현 제거 ("적절히", "최적화" 등)
  • 구체적인 수치와 기준 제시
2

Mutual Information 극대화

목표와 강하게 연결되고 서로 독립적인 정보 제공

  • 중복되지 않는 새로운 정보
  • 작업과 직접적으로 관련된 컨텍스트
3

정보 이득 임계점 돌파

실행 전 충분한 정보 확보

임계점 75% - 더 많은 정보 필요

임계점에 도달하기 전에는 절대 실행하지 않습니다!

🔧 원칙 4-6: 실행 방법론

4

자료 접근성 보장

AI가 필요한 모든 자료에 최소 노력으로 접근 가능

  • 모든 관련 파일 직접 읽기
  • Import 관계 명확히 파악
  • 도메인 간 인터페이스 문서화
5

순차 처리

→
분석
→
계획
→
실행
6

컨텍스트 엔트로피 순서

큰 덩어리에서 작은 것으로 좁혀가기 (Fitts's Law)

  • 전체 구조 파악 → 세부사항
  • 디렉토리 → 파일 → 함수

✅ 원칙 7-9: 품질 관리

7

Cascading Entropy 방지

단계가 진행될수록 엔트로피가 증폭되는 것을 방지

한 번에 4-5개 파일만 수정 → 커밋 → 다음 작업

8

교차 검증

9

발산 최소화

엔트로피를 증가시키는 요소 제거

  • 모호한 지시 제거
  • 오버엔지니어링 방지
  • 트리거 단어 사용 금지 ("적절히", "최적화" 등)

🎯 원칙 10-11: 최적화와 효율성

10

단일 진실의 원천 (Single Source of Truth)

모든 정보의 기준점을 하나로 유지

코드가 진실이다!

문서나 주석이 아닌 실제 코드를 기준으로 삼습니다

  • 주석과 코드가 다르면 → 코드를 믿기
  • 문서와 구현이 다르면 → 구현을 따르기
11

정보 이득 기반 작업 분배

각 작업을 가장 효율적인 주체에게 할당

작업 유형 최적 수행자 이유
반복적 코드 생성 AI 빠르고 일관성 있음
비즈니스 로직 설계 인간 도메인 지식 필요
테스트 코드 작성 AI + 인간 교차 검증 효과

📈 I/O Ratio - 입출력 비율의 중요성

INPUT : OUTPUT = 20 : 1
입력이 출력보다 20배 이상 많아야 안전

❌ 나쁜 예

입력: 100 토큰
출력: 500 토큰

→ AI가 추측으로 채움

✅ 좋은 예

입력: 2000 토큰
출력: 100 토큰

→ 정확한 결과

  • 실행 단계: 충분한 컨텍스트 확보 후 구현

🏗️ 레거시 코드 대응 전략

  • 높은 엔트로피 (일관성 부족)
  • 문서와 코드의 불일치
  • 구 코드와 신 코드의 혼재
  • 패치에 패치를 더한 흔적

대응 방법

  1. 휴먼 파워 활용: 비즈니스 목적, 아키텍처 패턴 파악
  2. 타협적 격리: Facade 패턴으로 인터페이스만 노출
  3. 점진적 수리: 중요 부분부터 단계적 개선

프로젝트 바운더리 설정

1. WHY - 비즈니스 목적

왜 이 프로젝트를 하는지 명확히 정의

2. WHAT - 목표 설정 (SMART)

  • Specific (구체적)
  • Measurable (측정 가능)
  • Achievable (달성 가능)
  • Realistic (현실적)
  • Time-bound (기한 설정)

3. HOW - 제약 조건

  • 예산 범위 (최소-최대)
  • 사용 가능한 도구, 스택, 인원
  • 코드 품질 수준 및 패턴
